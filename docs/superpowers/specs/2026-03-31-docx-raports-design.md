# Генерація рапортів у .docx — специфікація

## Проблема

Існуюча PDF-генерація вимагає від користувача заповнювати форму з персональними даними (ПІБ, звання, підрозділ) прямо в додатку. Це незручно і потенційно небезпечно — чутливі дані обробляються в браузері. Користувачі хочуть завантажити готовий документ і заповнити його самостійно в Word/Google Docs.

## Рішення

Замінити PDF-генерацію на .docx шаблони з плейсхолдерами для ручного заповнення. Кожен документ містить:
- Сторінка 1: рапорт/скарга з плейсхолдерами `{ПІБ}`, `{ЗВАННЯ}` тощо
- Сторінка 2+: лист-підказка (правова підстава, скрипт для подачі, що робити якщо не приймають)

## Архітектура

### Потік

1. AI відповідає → `templateDetector` визначає тип рапорту
2. Кнопка "📄 Завантажити рапорт (.docx)" з'являється під відповіддю (без форми)
3. Клік → `docxGenerator.ts` завантажує .docx шаблон, підставляє дату через `docxtemplater`, віддає Blob
4. Користувач відкриває .docx в Word/Google Docs і заповнює плейсхолдери

### Компоненти

#### templates/docx/*.docx — шаблони документів

6 .docx файлів, оформлених за Наказом МОУ №40:

| ID | Тип | Опис | Правова підстава |
|----|-----|------|-----------------|
| raport-nevyplata | рапорт | Невиплата грошового забезпечення | Постанова КМУ №704, Наказ МОУ №260 |
| raport-vidpustka | рапорт | Надання відпустки | ЗУ «Про соціальний і правовий захист», Положення про проходження служби |
| raport-zvilnennya | рапорт | Звільнення з військової служби | ЗУ «Про військовий обов'язок», Положення про проходження служби |
| raport-rotatsia | рапорт | Ротація | Положення про проходження служби, ЗУ «Про військовий обов'язок» |
| raport-vlk | рапорт | Направлення на ВЛК | Положення про ВЛК у ЗСУ |
| skarga | скарга | Скарга на дії командира | ЗУ «Про соціальний і правовий захист», ст. 7¹ |

**Структура кожного .docx:**

Сторінка 1 — рапорт:
- Шапка: кому (посада, звання, ПІБ командира — плейсхолдер)
- Від кого: посада, звання, ПІБ — плейсхолдер
- Заголовок: РАПОРТ / СКАРГА
- Основний текст з посиланнями на закони (конкретні статті/частини)
- Дата (підставляється автоматично), підпис (плейсхолдер)

Сторінки 2+ — ПІДКАЗКА (не для подачі):
1. **Мета рапорту** — 2-3 речення
2. **Правова підстава** — накази/закони з номером, датою, цитатою статті
3. **Що говорити при подачі** — короткий усний скрипт
4. **Якщо не приймають рапорт:**
   - Вимагати письмову відмову
   - Надіслати рекомендованим листом з описом вкладення
   - Подати скаргу вищому командуванню
   - Звернутися на гарячу лінію МОУ (1512) або до військової прокуратури
5. **Відповіді на можливі питання** — 3-5 типових заперечень та відповідей з посиланням на закон

Плейсхолдери в шаблонах: `{ПІБ}`, `{ЗВАННЯ}`, `{ПІДРОЗДІЛ}`, `{ПОСАДА}`, `{ДАТА}`, `{ПІБ_КОМАНДИРА}`, `{ЗВАННЯ_КОМАНДИРА}`, `{ПОСАДА_КОМАНДИРА}` та специфічні для кожного типу.

#### client/src/services/docxGenerator.ts — генерація .docx

```ts
async function generateDocx(templateId: string): Promise<Blob>
```

- Завантажує .docx з `/templates/docx/{templateId}.docx` через fetch
- Розпаковує через PizZip
- Підставляє `{ДАТА}` (поточна дата) через docxtemplater
- Повертає Blob для завантаження

#### client/src/services/templateDetector.ts — розширення

Додати патерни для нових типів:
- `raport-zvilnennya`: 'звільнен', 'демобіліз', 'закінч.*служб'
- `raport-rotatsia`: 'ротаці', 'заміна на позиці'
- `raport-vlk`: 'ВЛК', 'влк', 'лікарськ.*комісі', 'придатн.*служб', 'медичн.*огляд'

#### client/src/components/Chat.tsx — UI зміни

- Видалити DocGenerator (форму з полями)
- Замінити на кнопку "📄 Завантажити рапорт (.docx)" / "📄 Завантажити скаргу (.docx)"
- Клік → `generateDocx(templateId)` → автозавантаження

### Що видаляємо

- `client/src/components/DocGenerator.tsx` — форма з полями
- `templates/*.json` — старі JSON шаблони (raport-nevyplata.json, raport-vidpustka.json, skarga.json)
- Залежності: `pdf-lib`, `@pdf-lib/fontkit` з package.json
- З `pdfGenerator.ts` — функцію `generatePdf()` (залишаємо `exportChatToPdf()` для експорту бесіди)

### Що додаємо

- Залежності: `docxtemplater`, `pizzip`
- `templates/docx/` — 6 .docx файлів
- `client/src/services/docxGenerator.ts` — генерація
- Нові патерни в `templateDetector.ts`

## Тестування

- Unit тести для `docxGenerator.ts`: генерація повертає валідний Blob
- Unit тести для `templateDetector.ts`: нові патерни розпізнаються
- Перевірка що всі 6 .docx шаблонів існують і відкриваються
- Перевірка що `{ДАТА}` підставляється коректно
- Існуючі тести `exportChatToPdf` не ламаються

## Обмеження

- .docx шаблони — бінарні файли в git, ускладнюють diff/review
- docxtemplater у free версії не підтримує вставку зображень (не потрібно для рапортів)
- Користувач повинен мати Word/Google Docs/LibreOffice для редагування
