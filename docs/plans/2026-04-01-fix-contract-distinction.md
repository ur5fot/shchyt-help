# Виправити помилку з контрактами до/під час воєнного стану

## Overview
AI дає неправильну відповідь на питання "контракт укладений під час воєнного стану закінчився — що робити?". Відповідає "чекайте кінця війни", хоча пп. «ж» п.3 ч.5 ст.26 дає право на звільнення при закінченні контракту, укладеного ПІД ЧАС воєнного стану. Це системна проблема — AI не розрізняє правові режими.

Причини: 1) system prompt містить хибне узагальнення "контракти продовжуються автоматично"; 2) чанк з пп. «ж» може не потрапити в top-8 через слабкі keywords; 3) follow-up запити не враховують контекст попередньої відповіді.

## Context
- System prompt: `server/src/prompts/system.ts` — рядок ~7 містить "Контракти продовжуються автоматично на період воєнного стану" без розрізнення
- Критичний чанк: `laws/zminy-mobilizatsiya-2024.json`, chunk ID `зміни-щодо-мобілізац-st27-ch5-d7-p3` — містить пп. «ж» п.3 ч.5 ст.26 з правом на звільнення
- Пошук: `server/src/services/lawSearch.ts` — top-K = 8, синоніми, hybrid search
- Follow-up: `server/src/routes/chat.ts` — рядок ~126-143, query expansion для follow-up
- Eval: `eval/golden-set.json` — golden test set

## Technical Details

### Нове правило в system prompt
```
УМОВНЕ РОЗМЕЖУВАННЯ НОРМ: Багато норм військового права мають РІЗНІ правила залежно від умов. ЗАВЖДИ перевіряй та чітко розмежовуй:
- Коли укладено контракт: ДО воєнного стану vs ПІД ЧАС воєнного стану
- Період: мирний час vs особливий період vs мобілізація/воєнний стан
- Статус: контрактник vs мобілізований vs строковик vs офіцер
- Підстава призову: добровільно vs за мобілізацією
Якщо в наданому контексті є норми з різними умовами — ОБОВ'ЯЗКОВО вкажи усі варіанти та поясни, який застосовується до ситуації користувача.
```

### Query expansion для follow-up
```ts
const статтіЗВідповіді = останняВідповідь.content.match(/стаття\s+\d+|частина?\s+\d+|пункт\s+\d+|ст\.\s*\d+/gi);
if (статтіЗВідповіді) {
  контекстнийЗапит += ' ' + статтіЗВідповіді.join(' ');
}
```
