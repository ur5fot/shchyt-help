# Ієрархічне розбиття великих чанків у splitLargeChunks

## Overview
AI змішує підпункти з різних пунктів статті 26 ч.5 — наприклад, пп.«ж» п.1 (мирний час) та пп.«ж» п.3 (воєнний стан). Причина: `splitLargeChunks()` робить плоске розбиття — знаходить ВСІ маркери в одному проході, контекст пункту втрачається. Чанки `st26-ch5-ж` і `st26-ch5-ж-d2` мають однаковий `part: "Частина 5, пп.ж"`.

Потрібно дворівневий split: спочатку по цифрових маркерах (пункти), потім по літерних (підпункти). Результат: `part: "Частина 5, п.1, пп.ж"` vs `part: "Частина 5, п.3, пп.ж"`.

## Context
- `scripts/parse-law.ts` — `splitLargeChunks()` (рядки ~398-451), `МАКС_РОЗМІР_ЧАНКА = 2000`
- `parseArticleBased()` створює один чанк на частину статті (напр. вся ч.5 ст.26 = один чанк >3000 символів)
- `splitLargeChunks()` потім ріже цей чанк по підпунктах — але плоско, без збереження ієрархії
- Регекс зараз: `/\s(\d+\.\d+\.|\d+\)\s|([а-яіїєґ])\)\s)/g` — шукає і цифрові, і літерні маркери одночасно
- Це впливає на всі закони з ієрархічною структурою (ст.26, ст.20-2, ст.24 закону 2232-XII; аналогічні статті в інших законах)

## Technical Details

**Поточна проблема (плоский split):**
```
Вхід: "Підстави звільнення: 1) у мирний час: а) закінчення контракту; б) здоров'я; 2) під час ВС: а) вік; б) здоров'я;"
                                                                                                          
Вихід (зараз):       пп.1, пп.а, пп.б, пп.2, пп.а (d1), пп.б (d1)  — ієрархія втрачена
Вихід (потрібно):    п.1, п.1/пп.а, п.1/пп.б, п.2, п.2/пп.а, п.2/пп.б  — ієрархія збережена
```

**Формати ID:**
```
Зараз:    st26-ch5-ж      та  st26-ch5-ж-d2     (однаковий part)
Потрібно: st26-ch5-1-ж    та  st26-ch5-3-ж      (різний part: п.1 vs п.3)
```
