# SHCHYT MVP — повна реалізація проекту

## Огляд
Реалізація AI-асистента з прав військовослужбовців ЗСУ за специфікацією `shchyt-AGENTS.md`.
Локальний веб-додаток: чат з питаннями → відповіді з цитатами законів → генерація рапортів/скарг у PDF.

## Контекст
- Специфікація: `shchyt-AGENTS.md` — повний опис архітектури, коду, промптів
- Проект порожній — все створюється з нуля
- Стек: React + Vite + TypeScript + Tailwind (клієнт), Express + TypeScript (сервер), Claude API, pdf-lib
- Мова проекту: українська (коментарі, UI, документація)

## Підхід до розробки
- **Тестування**: TDD — спочатку тести, потім реалізація
- **Тестовий фреймворк**: Vitest (єдиний для клієнта і сервера)
- Кожен таск завершується зеленими тестами
- Малі, сфокусовані зміни
- Оновлювати цей файл при зміні скоупу

## Стратегія тестування
- **Unit-тести**: обов'язкові для кожного таску (Vitest)
- **E2E-тести**: Playwright для основного флоу чату (після Task 7)
- Тести перевіряють як успішні, так і помилкові сценарії

## Технічні деталі

### Формат запиту/відповіді API
```
POST /api/chat
Request:  { "message": "текст питання" }
Response: { "answer": "текст відповіді", "sources": [{ "law": "назва закону", "article": "Стаття X, Частина Y", "sourceUrl": "https://..." }] }
```

### Структура JSON законів
```json
{
  "title": "повна назва закону",
  "short_title": "скорочена назва",
  "source_url": "URL на zakon.rada.gov.ua",
  "last_updated": "YYYY-MM-DD",
  "chunks": [{ "id": "...", "article": "...", "part": "...", "title": "...", "text": "...", "keywords": [...] }]
}
```

### Вагові коефіцієнти пошуку
- Збіг по keywords: +3
- Збіг по тексту статті: +1
- Збіг по назві статті: +2
