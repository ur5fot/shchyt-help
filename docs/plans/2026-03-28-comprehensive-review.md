# Комплексне ревью проекту SHCHYT

## Overview
Системний аудит проекту з усіх сторін: безпека, якість AI-відповідей, пошук законів, UX, генерація PDF, якість коду, тести. Результат — конкретні виправлення, впорядковані за критичністю.

## Context (from discovery)
- Проект: AI-асистент з прав військовослужбовців ЗСУ
- Стек: React + Vite (клієнт), Express + TypeScript (сервер), Claude API, pdf-lib
- 22 тест-файли (~1350+ рядків), TypeScript strict mode
- 3 закони в базі, 3 шаблони документів
- Знайдені проблеми: відсутність rate limiting, немає ESLint, немає мінімального порогу пошуку, дублювання констант, немає логування

## Development Approach
- **Testing approach**: Regular (code first, then tests)
- Кожен таск — конкретне виправлення або група пов'язаних виправлень
- **CRITICAL: кожен таск ОБОВ'ЯЗКОВО включає нові/оновлені тести**
- **CRITICAL: всі тести мають проходити перед початком наступного таску**
- **CRITICAL: оновлювати цей план при зміні скоупу**

## Testing Strategy
- **Unit tests**: обов'язкові для кожного таску
- Запускати: `npm test` (root workspace — запускає тести клієнта і сервера)

## Progress Tracking

- Нові знахідки додавати з ➕ префіксом

## Technical Details

### Rate Limiting
- Бібліотека: `express-rate-limit`
- Ліміт: 20 запитів/хвилину на IP (для локального використання — м'який ліміт)
- Повертає 429 з повідомленням українською

### Пошук законів — нормалізація
- Базовий підхід: видалення типових українських закінчень (-ти, -ся, -ою, -ів, -ам тощо)
- Карта синонімів: 10-15 найчастіших пар для військової тематики

### Структуроване логування
- Формат: JSON в production, pretty в development
- Рівні: info (запити), warn (маргінальні результати), error (збої API)
