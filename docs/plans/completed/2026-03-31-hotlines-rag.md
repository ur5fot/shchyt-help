# База гарячих ліній для RAG

## Overview
Додати базу контактів гарячих ліній (телефони, email, адреси) як RAG-чанки, щоб AI-асистент міг рекомендувати конкретні контакти у відповідях коли це доречно. Контакти зберігаються у тому ж форматі що й закони — JSON з чанками в `laws/hotlines.json`.

## Context
- Існуючі закони: JSON файли в `laws/` з чанками (id, article, text, keywords)
- Завантаження: `laws/index.ts` автоматично читає всі JSON з `laws/`
- Індексація: `server/src/services/vectorStore.ts` — LanceDB cosine similarity
- System prompt: `server/src/prompts/system.ts` — інструкції для Claude
- Гібридний пошук: `server/src/services/lawSearch.ts` — keyword + vector + HyDE + re-ranking

## Development Approach
- **Testing approach**: Regular (код спочатку, потім тести)
- Формат чанків ідентичний існуючим законам — мінімальні зміни коду
- Контакти групуються тематично: кожна організація/лінія = окремий чанк
- System prompt оновлюється щоб Claude знав про контакти

## Testing Strategy
- **Unit тести**: валідація формату JSON, перевірка що чанки завантажуються, пошук знаходить контакти
- **Інтеграційні**: eval тест що Claude видає контакт на відповідне питання

## Technical Details

### Формат `laws/hotlines.json`
```json
{
  "title": "Гарячі лінії та контакти для військовослужбовців",
  "short_title": "Гарячі лінії",
  "source_url": "internal://hotlines",
  "document_id": "Довідник контактів",
  "last_updated": "2026-03-31",
  "chunks": [
    {
      "id": "hotlines-mou",
      "article": "Гаряча лінія Міністерства оборони України",
      "part": "",
      "title": "Контактний центр МОУ",
      "text": "Телефон: 0-800-500-442 (безкоштовно)...",
      "keywords": ["МОУ", "міністерство", "оборони", "гаряча", "лінія", "скарга", "звернення"]
    }
  ]
}
```

### Пошук контактів
Контакти знаходяться через існуючий гібридний пошук — keywords забезпечують keyword match, vector embeddings — семантичний пошук. Додаткова логіка не потрібна.
