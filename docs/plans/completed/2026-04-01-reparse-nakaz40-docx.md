# Перепарсити Наказ №40 з актуального .docx (зі змінами до 27.02.2026)

## Overview
Поточний JSON наказу №40 в базі застарілий — спарсений з PDF оригіналу від 31.01.2024. В корені проекту лежить актуальний .docx з усіма 6 наказами-змінами (41 зміна до 27.02.2026). Потрібно перепарсити наказ з .docx, замінити JSON, переіндексувати LanceDB.

## Ключові різниці поточного JSON vs актуального .docx
1. **Відсутній Розділ 14** (Порушення законодавства про НАФ) — повністю пропущений
2. **41 зміна** з 6 наказів (№346/2024, №441/2024, №183/2025, №497/2025, №2/2026, №77/2026) — відсутні
3. **Конкретні пропущені пункти:** п.2.3.4 (посадовий оклад), п.2.3.5 (винагорода за бойові), п.2.7.11 (обмежувальні грифи)
4. `last_updated` має бути `2026-02-27`, а не `2024-01-31`

## Context
- Актуальний .docx: `ГК ЗСУ №40 від 31.01.2024 (зі змінами від 27.02.2026).docx` (в корені проекту)
- Поточний JSON: `laws/наказ-40-діловодство-зсу.json` (630 чанків, застарілий)
- Парсер: `scripts/parse-nakaz40.ts` — парсить plain text, потрібно адаптувати для .docx або витягти текст з .docx
- Залежності: `pizzip` вже є в проекті (для docxtemplater) — можна використати для читання .docx
- Eval: 4 питання з категорії "діловодство" в `eval/golden-set.json`

## Technical Details

### Витягування тексту з .docx
```ts
import PizZip from 'pizzip';
import fs from 'fs';

const docx = fs.readFileSync('path/to/file.docx');
const zip = new PizZip(docx);
const xml = zip.file('word/document.xml')?.asText() || '';
// Strip XML tags, preserve paragraph breaks
const text = xml.replace(/<w:p[^>]*>/g, '\n').replace(/<[^>]+>/g, '').trim();
```
